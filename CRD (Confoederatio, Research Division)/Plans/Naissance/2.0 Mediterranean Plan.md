- **Brush Modes** should be refactored so that their logic is entirely dependent on `common/` files which can be easily scripted.
- **Hierarchy** should be simplified so it doesn't have thousands of polygons:
	- **Groups** should purely be for organisation purposes in the Hierarchy.
	- **Layers** should be refactored as <u>Tables</u> which are alphabetically sorted with pagination, at the top sit tabs (by unique entity type) which open typed <u>Sheets</u> within the Table. These tables are similar to **CRUD** interfaces.
	  
	  This allows Layers to handle thousands of entities easily. Behaviour can also be scripted per layer.
	- Geometries in Layers can have **Tags**, which determine the query filters applicable to each of them. Any extant groups underneath a **Layer** will automatically be converted to **Tags** for backward-compatibility purposes.
- `DALS.Timeline.parseAction` processing, as well as render queries should be offshored to a Worker that can stream Geometries to render.
