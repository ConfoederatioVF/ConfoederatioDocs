It is the most festive time of year, or would be if we were not so busy dispelling the _Ghost of Infrastructure Past_. Just yesterday, we received news that our moderation bot for some of our social media servers, **Snow Zed**, had a security flaw which meant that it had to be taken down for inspection and repairs, which we are still working on.

A minor hiccup, but Snow Zed's code is from years past, and such legacy flaws and oversights riddle our workflows today, which we have spent all of 2025 attempting to fix. Plans are yearly, naturally, so Confoederatio25 has given way to Confoederatio26.

And what makes up **Confoederatio26**? Why, the transformation of Confoederatio into an indie data science studio of sorts.
## Timetable

All dates are prospective and given as deadlines, rather than when they commence.

- 31 January 2026: [[Vercengen]] 1.0 Release
	- ve.Graph
	- ve.NodeEditor
	- ve.NodeEditorDatatype
- 31 July 2026: [[Naissance HGIS]] 1.0 Release
- 31 August 2026: Web infrastructure work
	- https://confoederatio.org/ modernisation work
	- Confoederatio Docs completeness
	- Naissance HGIS data portal/livemap capability
- 31 December 2026: Finish database polish work
	- Stadestér 1.1 (currently 1.0)
	- Eoscala 1.5 (currently 1.2)
	- Velkscala 1.0 (currently 0.7)
	- Atlas 1.0, intermittent work on Atlas via Naissance HGIS

It is naturally all about leverage. We first intend to work on our software engine (Vercengen), then use said software engine to build an integrated research environment (Naissance HGIS) to which we can port existing models/database code and ensure their self-consistency and reproducibility.

Much of this work has already been done, and Vercengen has only 3 components outstanding before it is available for a 1.0 release, and we will likely never have to touch it again apart from some minor improvements for the rest of the development cycle of Naissance HGIS, which will bring it to capabilities maturation.

None of this can be rushed, of course. That was the mistake of yesteryear. As a result, we have decided to devote long time horizons between these projects that will allow for them to be completed and polished to an acceptable degree to be adopted by others outside Confoederatio. 

We do not expect that if we build it, they will come, since we will be so busy building that we will not be marketing. But flash is not the same thing as polish. Indeed, they are opposites by definition.
